    Core Layer

    The last layer in the layered architecture is the Core Layer. Core Layer implements core application logic, such as State & Data manipulation, Business services and Data Source Access services (outside world communi-cation). NgRx framework is used for state management, this is the place to put our state definition, actions and reducers.

    [Angular mapping]

    In our examples we are modeling state with BehaviorSubjects. We can encapsulate it in any convenient state class. Below, you can find JsonViewerState example from the core layer.

    @Injectable()
export class JsonViewerState {
  private jsonData$ = new BehaviorSubject<string>('Hello from Core Layer - No Json loaded');
  private postData$ = new BehaviorSubject<PostDBO[]>(null);
  private commentData$ = new BehaviorSubject<CommentDBO[]>(null);
  private profileData$ = new BehaviorSubject<ProfileDBO>(null);

  constructor() {}

  public getJsonData$(): Observable<string> {
    return this.jsonData$;
  }

  private updateText() {
    let text: string;
    let results = [ this.postData$.getValue(),
                    this.commentData$.getValue(),
                    this.profileData$.getValue()];
    text = JSON.stringify(results, undefined, 2)
    this.jsonData$.next( text );
  }

  public addPosts(posts: PostDBO[]) {
    this.postData$.next( posts );
    this.updateText();
  }

  public addComments(comments: CommentDBO[]) {
    this.commentData$.next( comments );
    this.updateText();
  }

  public addProfile(profile: ProfileDBO) {
    this.profileData$.next( profile );
    this.updateText();
  }
}

    In the Core Layer, we implemented HTTP queries in the form of class providers. This kind of class could have API or DSA (stands for: data source access) name postfix. API services have only one responsibility - it is just to communicate with API endpoints and nothing else. Please avoid any caching, logic or data manipulation here. The manipulation of the provided data from the Data Source Access (DSA) services will be done in separate Business Services components and/or State & Data Business Objects management (STATE & DBO) components.

    A simple example of API/DSA service can be found below.

    @Injectable()
export class JsonAccessManagerDSA {
  // User the json-server for simulaing an REST JSON API server:
  // <https://www.npmjs.com/package/json-server>
  // install: npm install -g json-server
  // start: "json-server --watch db.json" in a terminal
  private baseUrl: string = <http://localhost:3000>;

  constructor(private httpClient: HttpClient) {
  }

  public loadPosts$(): Observable<object> {
    return this.httpClient.get(this.baseUrl + '/posts');
  }

  public loadComments$(): Observable<object> {
    return this.httpClient.get(this.baseUrl + '/comments');
  }

  public loadProfile$(): Observable<object> {
    return this.httpClient.get(this.baseUrl + '/profile');
  }
}

    Example of a Business Services

    @Injectable()
export class JsonManagerService {
  constructor(private jsonAccessManagerDSA: JsonAccessManagerDSA, private jsonViewerState: JsonViewerState,) {}

  loadJsonData() {
    this.jsonAccessManagerDSA.loadPosts$().subscribe({
        next: (result$: PostDBO[]) => {
          this.jsonViewerState.setPosts(result);
        },
        error: (err: any) => {
          console.log(err);
        },
        complete: () => {
          console.log('complete');
        }

    Example of a DBO

    export class PostDBO {
    public id: string;
    public postId: string;
    public comment: string;

    public constructor(init?:Partial<CommentDBO>) {
        Object.assign(this, init);
    }
}

    Data Business Objects (DBOs like PostDBO) will be defined in the Core Layer, too. These data objects represent the internal data model of the UI and they will be used to define a state or as transport containers between the layers. This kind of class could have DBO (stands for: data business object) name postfix The Core Layer will be the place for any validators, mappers or more advanced use-cases that require manipulating many slices of our UI state.

    We have covered the topic of the Abstraction Layers in our frontend application. Every layer has its well- defined boundaries and responsibilities. We also defined the strict rules of communication between layers. This all helps to better understand and reason about the system over time as it becomes more and more complex.
