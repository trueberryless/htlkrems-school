    Toolbar

    Design

    The toolbar represents the application and provides information and actions.

    Usage

    The toolbar reflects the branding as well as the purpose of the view through its title. Furthermore, it provides actions and can hold a breadcrumb to support navigation. Therefore, it is an important component, which should be provided with every app. It can transform into a contextual action bar(see 'Behavior').

    Use When

    • frequently used commands and actions shall be provided being permanently displayed for quick access

    Use Alternatively

    • buttons in the particular views for actions and commands, which are applicable only in these views
    • drop down menus in case there are many commands and actions

    Anatomy

    • Bar - a rectangular, solidely colored container, extending the full width of the layout, displaying elements like icon, action items, title. The container can be vertically extended from regular size to a larger one to make it more prominent - usually reserved for promotional applications.
    • Navigation icon (optional) - optionally an icon displayed on the left of the bar representing either
      • a drop-down-menu with global actions ("Hamburger Menu")
      • a button (back arrow) for returning to the previous view
      • a button (up arrow) for navigating upwards in the apps hierarchy.
    • Screen Title - is placed on the right hand of the navigation icon and used to show either
      • application title
      • title of the view, which is currently shown

    Titles are normally left aligned. This can vary depending on the OS (mobile, tablet). For title definition see Wording. If the title is too long, it must not be truncated, but can be wrapped, supposed it is placed on a vertically enlarged ("prominent") bar.

    • Actions (optional) - one or more context related action items represented as buttons that help work on a process performed by the application. Placed on the right end of the bar, sorted by their relevance / frequence of use (most used action on the left). Action items move to the overflow menu on the right end of the bar, if there is not sufficient place for them(less used items first).

    Placement

    The toolbar is usually placed on the top of the screen.

    Behavior

    • An top app bar can change to a contextual action bar (color change) as soon as one element in the view below has been selected. It shows
      • a close icon,
      • the number of selected items,
      • actions, applicable for the selection and
      • an overflow menu, if necessary.

    The contextual action bar remains active until an action is taken or it is dismissed and will transform back to the top app bar afterthat (color change). - Scrolling up in the view below the app bar - hides the bar or - reduces an enlarged ("prominent") bar to a regular one.

    Scrolling down will show the bar in full size. - Resizing the screen has a direct impact on the app bar, which will resize with it. Depending on the resulting space, actions are shown on the bar or in the overflow menu.

    The following basic implementation can be used to apply Sioux toolbar:

    app.module.ts

    import { NgModule } from '@angular/core';
import { HttpClient, HttpClientModule } from '@angular/common/http';
import { BrowserModule } from '@angular/platform-browser';
import { AppComponent } from './app.component';
import { BrowserAnimationsModule } from '@angular/platform-browser/animations';
import { SiOUXThemeSwitcherModule, ThemeNotifierService} from '@simpl/sioux-ng';
import {SiouxContentHeaderModule}  from '@simpl/sioux-ng';
import { ComponentInjectorService} from '@simpl/sioux-ng';
import { TranslateLoader, TranslateModule } from '@ngx-translate/core';
import { TranslateHttpLoader } from '@ngx-translate/http-loader';
import { MaterialAppModule} from './material.module';
import { AppRoutingModule } from './app-routing.module';
import { AngularSplitModule } from 'angular-split';
import { ButtonComponent } from './components/button/button.component';

export function HttpLoaderFactory(http: HttpClient) {
  return new TranslateHttpLoader(http);
}

@NgModule({
  declarations: [
    AppComponent,
    ButtonComponent
  ],
  imports: [
    AngularSplitModule,
    AppRoutingModule,
    BrowserModule,
    BrowserAnimationsModule,
    MaterialAppModule,
    HttpClientModule,
    SiouxContentHeaderModule,
    SiOUXThemeSwitcherModule,
    TranslateModule.forRoot({
      loader: {
        provide: TranslateLoader,
        useFactory: (HttpLoaderFactory),
        deps: [HttpClient]
      }
    })
  ],
  providers: [
     ComponentInjectorService,  ThemeNotifierService  
    ],
  entryComponents: [],
  bootstrap: [AppComponent]
})
export class AppModule { }

    app.component.ts

    import { Component, OnInit  } from '@angular/core';
import { TranslateService } from '@ngx-translate/core';
import { AppService, ContentHeaderService } from '@simpl/sioux-ng';

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.scss']
})
export class AppComponent implements OnInit {

  mataccenticons: string[];
  matprimaryicons: string[];
  title = 'SiOUX-WebApp';
  data: any[] = [];


  constructor(translate: TranslateService, public appService: AppService,
    private readonly _contentHeader: ContentHeaderService) {
    translate.setDefaultLang('en');
    translate.use('en');
  } 


  //this method is used to identify the icons and perform the click
  public performIconOperation(iconName) {
    if(iconName=="favorite") {
      console.log("favorite clicked");
    }
    if(iconName=="delete") {
      console.log("delete clicked");
    }
  }

  ngOnInit(): void {
    // Pass the Icon's which is to be placed in header
    this.matprimaryicons = ['favorite', 'delete'];
    this.mataccenticons = ['settings'];
    this._contentHeader.pagetitle.title('SiOUX Test Application');
  }

  //this method is executed, when the hamburger menu is clicked. 
  onOpenMenu(menu: any): void {
    console.log(menu);
  } 
}

    app.component.html

    <app-sioux-content-header [matIcons]="matprimaryicons" [matAccentIcons]="mataccenticons" (clickIcon)="performIconOperation($event)">
</app-sioux-content-header>

    Page Title

    The page title can be configured as follows:

    import { Component, OnInit } from '@angular/core';
import { ContentHeaderService } from '@simpl/sioux-ng';

@Component({
  selector: 'app-button',
  templateUrl: './button.component.html',
  styleUrls: ['./button.component.scss']
})
export class ButtonComponent implements OnInit {

  constructor(private readonly _contentHeader: ContentHeaderService
    ) {  }

  ngOnInit(): void {
    this._contentHeader.pagetitle.title('SiOUX Button');
  }
}

    User Defined Iconic Buttons

    The SiOUX toolbar provides the possibility to add user defined iconic buttons to the toolbar. They can be either of the standard toolbar icon color or of accent color. To use them, you need to define and initialize them within your class:

    matprimaryicons: string[];
mataccenticons: string[];

ngOnInit(): void {
  // Pass the Icon's which is to be placed in header
  this.matprimaryicons = ['favorite', 'delete'];
  this.mataccenticons = ['settings'];
}

    If you want to react on the click of the user defined buttons you need to implement the 'performIconOperation(iconName)' method:

      //this method is used to identify the icons and perform the click
  public performIconOperation(iconName) {
    if(iconName=="favorite") {
      console.log("favorite clicked");
    }
    if(iconName=="delete") {
      console.log("delete clicked");
    }
  }

    Additionally you must add them to you html-template:

    <app-sioux-content-header [matIcons]="maticons" [matAccentIcons]="mataccenticons" (clickIcon)="performIconOperation($event)">
</app-sioux-content-header>

    Theme Switch

    If you want to implement a theme switch within your application you first of all need to generate your application based style files. This can be done with the following commands within the "scripts" section of your package.json file:

    "scripts": {
    "ng": "ng",
    "theme:build:light": "sass --load-path=node_modules src/assets/themes/sioux-light-theme.scss src/assets/themes/angular/sioux-light-theme.css",
    "theme:build:dark": "sass --load-path=node_modules src/assets/themes/sioux-dark-theme.scss src/assets/themes/angular/sioux-dark-theme.css",
    "start": "npm run theme:build:light && npm run theme:build:dark && ng build && ng serve",
    "build": "ng build",
    "watch": "ng build --watch --configuration development",
    "test": "ng test"
  },

    Additionally you need to adapt the "styles" sections of your angular.json file accordingly:

     "styles": [
              "src/styles.scss",
              "src/assets/themes/angular/sioux-light-theme.css",
              "src/assets/themes/angular/sioux-dark-theme.css",
              "node_modules/@simpl/sioux-theme/fonts/theme-fonts.css"
            ],

    Add to your styles.scss file the following:

    @use './assets/themes/angular/sioux-dark-theme.css' as dark;
@use './assets/themes/sioux-dark-theme.scss';

    Additionally to the automatically generated css-files you need to include the following two scss-files to your asset/themes folder:

    • sioux-dark-theme.scss
    • sioux-light-theme.scss

    sioux-dark-theme.scss

    @use 'sass:map';
@use '../../../node_modules/@simpl/sioux-theme/themes/angular/scss/material-design/sioux-dark-theme.scss' as dark;
@use '../../../node_modules/@simpl/sioux-theme/themes/angular/scss/material-design/colors/dark-theme-colors.scss' as darkColor;

    sioux-light-theme.scss

    @use 'sass:map';
@use '../../../node_modules/@simpl/sioux-theme/themes/angular/scss/material-design/sioux-light-theme.scss' as light;
@use '../../../node_modules/@simpl/sioux-theme/themes/angular/scss/material-design/colors/light-theme-colors.scss' as lightColor;

    Component specific styling

    If you need component specific styling, the best is to add a component specific theme file which contains your mixins:

    Here is a sample content of button specific styling which uses Sioux specific colors:

    button.component-theme.scss

    @use 'sass:map';
@use '@angular/material' as mat;
@use '@simpl/sioux-theme/themes/angular/scss/material-design/common/theming.node' as node;

@mixin own-button($theme) {
    $functionalColors: node.get-functional-colors($theme);
    $functionalGreen: map.get($functionalColors, functionalGreen);
    $background: map.get($theme, background);
    $foreground: map.get($theme, foreground);
    $button: map.get($background, page-background);

    .mat-primary.mat-raised-button {
        background-color: red;
    }

    .mat-accent.mat-raised-button {
        background-color: $functionalGreen;
    }
}

    If you need to retrieve the sioux functional colors or the sioux supporting colors supporting colors you need to use the theming node file as in the sample above and then you can access it as follows:

    $functionalColors: node.get-functional-colors($theme);
$supportingColors: node.get-supporting-colors($theme);

    The resulting array can then be utilized by means of map.get.

    If you want to access the theme palette colors, you can achieve this as follows:

     $color-config: mat.get-color-config($theme);
    $primary-palette: map.get($color-config, 'primary');
    $accent-palette:  map.get($color-config, 'accent');
    $warn-palette:    map.get($color-config, 'warn');
    $is-dark-theme:   map.get($color-config, 'is-dark');

    .mat-primary.mat-raised-button {
        background-color: map.get($warn-palette, 500);
    }

    In the end all component specific theme files must be used in the theme-specific scss-files (sioux-light-theme.scss or sioux-dark-theme.scss):

    sioux-dark-theme.scss containing component specific styling

    @use 'sass:map';
@use '../../../node_modules/@simpl/sioux-theme/themes/angular/scss/material-design/sioux-dark-theme.scss' as dark;
@use '../../../node_modules/@simpl/sioux-theme/themes/angular/scss/material-design/colors/dark-theme-colors.scss' as darkColor;

//application-related theme files
@use '../../app/components/button/button.component-theme.scss' as button;


@include button.own-button(darkColor.$theme);

    API Reference for SiOUX Themepicker Icon (Toggle Theme)

    import {SiOUXThemeSwitcherModule} from '@simpl/sioux-ng';

    Directives

    ThemePickerIconComponent

    Selector: sioux-theme-picker-icon

    Exported as: ThemePickerIconComponent

    Properties

    Name Description
    tooltipText: string The text displayed in the tooltip

    Methods

    toogleTheme

    Switches the theme between sioux-light-theme and sioux-dark-theme

    API Reference for SiOUX Themepicker Menu (Select a Specific Theme)

    import {SiOUXThemeSwitcherModule} from '@simpl/sioux-ng';

    Directives

    ThemePickerMenuComponent

    Selector: sioux-theme-picker-menu

    Exported as: ThemePickerMenuComponent

    Properties

    Name Description
    themes: Theme[] To get the available themes
    currentThemes: Theme[] To get the current theme
    tooltipText: string The text displayed in the tooltip

    Methods

    setTheme

    Sets the selected theme

    Interfaces

    Theme

    Properties

    Name Description
    name: string name of the theme
    accent: string accent colour of the set theme
    primary: string primary color of the set theme
    warn: string warn color of the set theme
    background: string background color of the set theme
    foreground: string foreground color of the set theme
    isDark?: boolean true if the current theme is dark
    isDefault?: boolean true when the current theme is default
    displayName: string; Denotes the display name of the theme in the menu
