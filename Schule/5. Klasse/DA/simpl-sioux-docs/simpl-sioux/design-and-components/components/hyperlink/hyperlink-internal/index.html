    Hyperlink Internal

    Design

    An internal hyperlink is a reference to another location within the same application. It allows a better navigation through the website or an app and may appear as text, a graphic or both.

           

    Alias

    • Linked items

    Usage

    Use When

    • to help the user in accessing more information, i.e. reaching website, view, file, image and even dynamically created website
    • navigation inside application is required.

    Use Alternatively

    • a button, when
      • an execution of an action is required or
      • a dialog shall be invoked

    Anatomy

    • The internal hyperlinks are often simply built into the text.
    • Link labels should clearly indicate the result of activating the hyperlink. Avoid using
      • the words "Click here" to identify a link.
      • unnecessary navigational information (e.g. "Go to homepage").
    • For label definition see Wording.

    Placement

    Make sure that you place your internal link somewhere in the header, or upper body of the content rather than on the sidebar (or worse, in the footer).

    Behavior

    • By activating an internal hyperlink the linked location is opened in the current view
    • The link shows the address of the destination in a tooltip
    • The text of the internal link does not change its color after click ("visited").
    • By activating a back navigation button in the toolbar the first view is shown again.
