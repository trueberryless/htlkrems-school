    Iconography

    General

    Icons are graphical symbols that represent objects, functions and processes in a pictorial manner. Icons are shown directly on the screen background and are very economic and powerful tools as they avoid localization of screens and support a fast orientation on the screen. Each icon has a unique appearance and meaning. Consistent use of icons enables an efficient user interaction.

    There are different catogories of icons

    • Standard icons (file, view, edit, open, search...)
    • Engineering icons without domain-specific context (power flow, economic dispatching)
    • Symbols and graphical display elements for dynamic process states (e.g. breakers and transformers in graphical representations of energy networks)
    • Application icons for application header and desktop symbols

    Usage

    Icons are used to:

    • Enable worldwide compatibility: no translation is necessary
    • Support the users' orientation: provide a fast overview about available functions
    • Reduce learning efforts: once the users have understood the meaning of an icon they can apply it to other contexts

    Something to consider:

    • Users rely on secondary cues (color, shape etc.) to distinguish groupings and locate relevant information quickly and easily.
    • Icons are better preattentive features than colors: users are faster in detecting indicators / locating an item when icons are used instead of color. The reason might be that color is more ambiguous than an icon; users have to infer that red means a decrease, which may take slightly longer than to grasp the meaning of an upward arrow.
    • However the best user performance can be achieved by combining unique icons with color. Using only one method of differentiating items, lessens the likelihood to notice small differences (e.g. color blind people).

    NOTE: Consider data-ink-ratio (figures) and signal-to-noise-ratio

    Rules to be followed

    • Do not design icons on your own.
    • Do not copy icons from the internet or other sources as this will have legal consequences for Siemens.
    • You can either use the Google material icons which are available in different format as well as web font or you can use the icons which are provided together with SiOUX.
    • An overview about the icons which have been created specifically for SiOUX can be found on the SiMPL Icons page below the following link: https://simpl.code.siemens.io/simpl-icon-demo/#/sioux/icons
    • Here you can find all icons sorted by available categories including different tags. A search, a direct copy and the possibility to download the complete set is available.
    • Additionally all SiOUX icons are provided as easy to use web font.
    • An instruction how to use the icons can be found on the icons details tab.
    • If no icon is suitable for the specific application case, request an adequate icon from your usability engineer.

    How to find an appropriate icon

    Before using an icon from the Google Material Design or the SiMPL icons web page in your project, you need to check your choice with the Usability Engineer in order to:

    • maintain a consistent icon use within the project
    • maintain a consistent icon use with regard to other SI applications

    How to request a new Icon

    In case that no appropriate icon could be found, the icon design process has to be initiated by the developer. To start this process the developer needs to provide the completed icon request template to your usability engineer. The usability engineer authorizes and coordinates the icon design and will ensure that the new icon is accessible via Git.

    Which icon is needed

    The necessary information for icon design is derived by developers. They may be supported by the usability engineer.

    All icons must be comprehensible and easy for users to learn and recognize. To reach that goal, the functionality represented by the icon needs to be analyzed and understood very well. That also contains information about the context in which the icon is used and, related functionalities. Take your time for analyzing the icons function.The fact that icons consist only of a few pixels, does not mean that it is little effort to design them properly.The following questions cover the most important aspects to understand the icons functionality. Without answers to these questions, icon design is not possible.

    Context

    • For which product is the icon needed?
    • Who will use the icon? E.g., a specific user role
    • When will it be used? E.g., the icons function is part of a typical workflow.
    • On which background will the icon be shown (provide a screenshot)?

    Functionality & Symbols

    • What kind of icon is needed?
    • Which kind of functionality does the icon represent?
    • Is the icon clickable? If so, what happens when it is clicked?
    • Which metaphor can be used to represent the icons functionality (for example, a disk for save)? Do the users associate a certain symbol with that function? Is there a standardized symbol for that function?
    • What are related but excluded functions (what should the icon not stand for)?
    • For the redesign of icons: What does the current icon represent? Why is the current icon not appropriate?

    Graphic Style

    • Are there any specific color requirements? E.g., the real-life object to be represented has a characteristic color.
    • Are there any restrictions concerning color depth or transparency?

    Which effort needs to be considered

    The effort for icon design must be included in the project plan. A rough estimation for creating a new icon is 4 hours. This effort includes the following tasks:

    • analysis of the icon
    • definition of an appropriate symbol which is in line with the DG icon language
    • visual design according to the DG icon style and project requirements (color depth etc.)
    • internal reviews
    • rework of the symbol

    Do not underestimate the effort for icon design. Especially the small size of icons makes it difficult to represent a complex functionality in a meaningful and recognizable way.

    Details

    Getting an Overview About the Available Google Material Icons

    An overview about the available icons can be found in the following link:
    https://material.io/resources/icons/?style=baseline

    The icons can be filtered by name, category and theme. For SiOUX the icons must use the theme "filled". The icons are provided as .svg or .png in diverse sizes (18dp, 24dp, 36dp, 48dp, iOS and Android).

    How to Use Google Material Icons

    In order to use the Google Material Icons you need to follow the description of Material Icons Guide:
    https://google.github.io/material-design-icons/

    For Web applications especially the description "Icon font for the web" is of relevance. Here you can find all information you need to integrate the icons as WOFF/WOFF2 into your application which facilitates the handling of the icons as the color and size can be set similar to the usage of a normal font.

    Getting an Overview About the SiMPL SiOUX Icons

    SiMPL SiOUX provides a demo web page where all available icons are listed:
    https://simpl.code.siemens.io/simpl-icon-demo/#/sioux/icons

    The icons are sorted by category and additionally tags are supplied which are used to filter the list of available icons in addition to its name. When clicking on any icon, the icon gets copied. Additionally it is possible to download the complete set of icons as .tgz.

    How to Use SiMPL SiOUX Icons

    In order to use SiMPL SiOUX Icons the following steps should be executed:

    • Install the package using:
    npm install --save @sioux/sioux-icons
    • Add the following lines to your main .scss file:
    $font-path: '@simpl/sioux-icons/dist/fonts';
@import '@simpl/sioux-icons/dist/style/simpl-sioux-icons.scss';
    • Use the icons by adding the relevant icon class. This should be the icon name identified in the SiMPL SiOUX demo web page. You need to add the class to the tag or any other element as follows:
    <mat-icon class="sioux-pq-evaluation"></mat-icon>

    Further details can be found in "https://code.siemens.com/em-dg/reuse/multiplatform/sioux/simpl-sioux-icons".

    How to Add New Icons to SiMPL SiOUX

    • Clone the repository "https://code.siemens.com/em-dg/reuse/multiplatform/sioux/simpl-sioux-icons" and add all the required dependencies:
    ## Clone the repository

git clone git@code.siemens.com:simpl/simpl-sioux-icons.git

## Change into project directory

cd simpl-sioux-icons

## Install all dependencies

yarn install

    • Add the new svg-icon to the simpl-SiOUX-icons\assets folder.

    • Describe your icon in the simpl-SiOUX-icons\src\sioux-icon-metadata.json file.

    • Build the fonts by means of yarn:

    yarn build
    • Check the result open the index.html file in the simpl-SiOUX-icons\dist folder.
    • Release the new version.

    To release the new version the subsequent steps need to be proceeded:

    • Update the version in the simpl-SiOUX-icons\package.json.
    • Update the changelog.
    • Commit everything by following the rules below:

    In general this should be similar to:

    feat: Add new "icon name"
    • Tag the version in code.siemens.
