    States

    General

    In SiOUX the following states exist:

    1) Hover
    2) Focus
    3) Pressed
    4) Dragged
    5) Selected
    6) Activated
    7) Disabled
    8) Read only

    Hover

    If the user places the mouse cursor over a control without clicking it, the control is hovered. This state exists for all interactive controls such as lists, buttons, cards,... In the table below you can find examples of hovered SiOUX controls:

    Control Example Dark Theme Example Light Theme
    Button
    Table
    Navigation Drawer

    Focus

    A control is in focused state, when the user selects the control with help of the keyboard or the mouse. All interactive controls can get focused. In the table below you can find examples of focused SiOUX controls:

    Control Example Dark Theme Example Light Theme
    Accent Button
    Text Field
    Radio Button

    Pressed

    When the user clicks or taps on a control it is pressed. All interactive controls can be pressed. In the table below you can find examples of pressed SiOUX controls:

    Control Example Dark Theme Example Light Theme
    Accent Button
    Tab
    Checkbox

    Dragged

    A component is in dragged state, when the user presses and moves the component. In the table below you can find examples of dragged SiOUX controls:

    Control Example Dark Theme Example Light Theme
    Card
    Dragged List Item
    Dragged Chip

    Selected

    If a user clicks or tabs a component or a part of the component, this component or component part is selected. In the table below you can find examples of selected SiOUX controls:

    Control Example Dark Theme Example Light Theme
    Card
    Dragged List Item

    Activated

    Controls in activated state give the user a hint which items are currently viewed. Controls are either activated per default or on user choice. In the table below you can find examples of activated SiOUX controls.

    Control Example Dark Theme Example Light Theme
    Navigation Drawer
    Chip
    Tab

    Disabled

    Controls in disabled state cannot be used. They are not interactive and not of importance for the user. In the table below you can find examples of disabled SiOUX controls:

    Control Example Dark Theme Example Light Theme
    Primary Button
    Chip

    Read Only

    Controls in read only state are not interactive. This means the content is not editable, but of importance for the user. In the table below you can find examples of read only state in SiOUX.

    Control Example Dark Theme Example Light Theme
    Table Grid
