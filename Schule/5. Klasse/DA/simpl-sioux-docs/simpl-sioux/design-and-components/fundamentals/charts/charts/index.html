    Charts

    Design

    Chart

    A chart is a graphical representation of data. It can represent tabular numeric data, functions or some kinds of qualitative structures. Different chart types can be used to depict the information given. Which type should be used depends on the given information.

    SiOUX does not provide an own charting library, but only the style for the following existing charting libraries:

    • Highcharts
    • ECHARTS

    Chart Types

    SiOUX currently supports the following chart types:

    Chart Use Highcharts ECHARTS
    Bar Chart To present few data over a nominal (e.g. countries, testing conditions, ...) or interval scale (e.g. time)
    Bubble Chart Is used to represent one to four dimensions. The first and second are represented by means of the coordinates. The third by color and the fourth by the size of the bubble
    Column Chart To present few data over a nominal (e.g. countries, testing conditions, ...) or interval scale (e.g. time)
    Area Chart To present data in different ways on a single chart
    Donut Chart Is used to show the proportions of categorical data and how partial elements add up to a total
    Line Chart - To display long data rows
    - To interpolate between data points
    - To extrapolate beyond known data values (forecast)
    - To compare different graphs
    - To find and compare trends (changes over time)
    - To recognize correlations and covariations between variables
    - If the X axis requires an interval scale
    - If the X axis requires an interval scale
    - To display interactions over two levels on the X axis
    - When convention defines meaningful patterns (e.g. a zigzag line)
    Stacked Column Chart To show the parts of multiple totals

    Samples for each chart type can be found in the library specific area.

    Details

    For SiOUX charts the following basic elements have been defined:

    Chart title

    Definition

    Each chart has a clearly descriptive title. Its function is to introduce into the topic at a glance.

    Solution Specification

    Do not use abbreviations within a charts title. An exception could be if they are commonly used by the target group of the particular application.

    Axes and Cells

    Definition

    The X axis (horizontal axis) and the Y axis (vertical axis) build together the main structure of a chart. There are labels at both axes which designate the scale of measurement or group of objects represented by the axis. By drawing horizontal/vertical lines from the several labels a grid is generated which consists of cells. The grid makes the structure more clearly-arranged.

    Solution Specification

    • Provide at least two axes along with a scale
      • X axis (the scale is placed at the bottom and is incremented from left to right)
      • Y axis (the scale is placed on the right hand side and is incremented from bottom to top)
    • Label the marker points along each axis; they need to form the point of origin for chart lines running perpendicular to the axis.
    • These chart lines define the chart cells, thus they are forming the matrix within the value indicators are plotted. The size of the cells depends on the intervals at which markers are placed on the X and Y axes.
    • Optionally a second Y axis can be included at the right of the chart with a different scale compared to the first Y axis.

    Labels

    Defintion

    After reading the chart title the viewer takes a look at the labelling of the axes. The axes are labeled with axis labels as well as marker point labels. Both are the basis for the understanding the chart's message.

    Solution Specification

    • Attach all labels to the outside edge of each axis of a chart.
    • Write labels in sentence-style, i.e. only the first letter on the first word of a label is capitalized; all other words are written in lower case, except for proper names and recognized abbreviations.

    Value Indicators

    Defintion

    Value indicators are the graphical elements, in the form of lines, bars, areas, series of dots or vectors that plot the actual values recorded within the structure of a chart.

    Solution Specification

    • Determine the type of the chart on the basis of the form (or combination of forms) of value indicator used in a chart.
    • Differentiate value indicators applying to different objects within one chart by color.
    • Layer the value indicators one over the other from front to back in the order in which they are listed in the legend in charts with multiple value indicators (e.g. lines) that intersect at various points.

    Legend

    Definition

    Charts containing more than one value indicator (line, bar, area, series of dots) are provided with a legend explaining what is signified by each value indicator. The purpose of the legend is

    • To explain to users how the various value indicators are visually differentiated.
    • To specify what data is represented by each value indicator.
    • To give users the possibility of showing/hiding individual value indicators (if appropriate).

    Solution Specification

    • Place the legend
      • if possible to the bottom of the chart. The legend items must be center aligned.
      • to the right of the chart (top line aligned with the top of the chart), with each legend item on a seperate row.

    Tooltips

    Definition

    Tooltips in charts contain explantory information on an individual or multiple value indicators. They appear when the mouse pointer is moved over the value indicator, and disappears when the mouse pointer is moved to a different position.
